INTERACTION FLOWS
═══════════════════════════════════════════════════════════════════════════

1. CLUSTER SWITCHING
   User clicks cluster in dropdown
   → Switch animation (spinner on selected cluster)
   → Backend API call to switch context
   → Success: Update current cluster badge, refresh pods
   → Error: Show error toast, revert to previous cluster

2. REFRESH CLUSTERS
   User clicks refresh button (↻)
   → Button shows spinner animation
   → Backend API call to re-fetch cluster list
   → Update dropdown with new cluster data
   → Preserve current selection if still available

3. MANAGE CLUSTERS
   User clicks "Manage Clusters" footer button
   → Close dropdown
   → Open Kubernetes Settings modal
   → Display kubectl configuration instructions

4. PERSISTENCE
   User switches to new cluster
   → Save context_name to localStorage
   → On page reload, check if saved context exists
   → If exists and accessible, auto-switch to it
   → If not accessible, use current kubectl context
