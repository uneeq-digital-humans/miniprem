# kiosk-ui Helm chart

Packages the **front door** of the UneeQ Dell all-in-one: the kiosk SPA plus the
Ingress that reverse-proxies every on-box service through a single host
(`digitalhuman.miniprem` / `localhost`).

```bash
helm upgrade --install kiosk-ui ./kiosk-ui -n uneeq
# override the host(s) / image as needed:
helm upgrade --install kiosk-ui ./kiosk-ui -n uneeq \
  --set image.tag=1.5.0 --set 'ingress.hosts={kiosk.acme.local}'
```

## What it deploys

| Resource | Purpose |
|---|---|
| `Deployment/digitalhuman-interface` | nginx serving the built Dell SPA |
| `Service/digitalhuman-interface:80` | the kiosk service the ingress targets |
| `Ingress/kiosk-ui` | `/` → kiosk, `/v1` → rag-adapter (served as-is) |
| `Ingress/kiosk-ui-proxies` | `/rag-admin`, `/host-admin`, `/monitor`, `/api/asr` (path stripped → backend) |

The proxy map (one front door → all services):

```
/            → digitalhuman-interface:80   (kiosk SPA)
/v1          → rag-adapter:8085            (LLM / conversation, OpenAI-compatible)
/rag-admin/* → rag-adapter:8085            (kiosk prompt + document admin)
/host-admin/*→ host-helper:8086            (GPU/STT/TTS/RAG health, audio, model pull, RAG scale)
/monitor/*   → miniprem-monitor:3001       (MiniPrem monitor)
/api/asr/*   → digitalhuman-asr:8000       (riva-ws-proxy — STT websocket; route off until image pullable)
```

## Image / Harbor note

`image.repository` defaults to `cr.uneeq.io/dell-isg-containers/digitalhuman-interface`.
That project is currently **Harbor-permission-blocked** for the `uneeq` robot, so
on the live box the SPA is served by `deploy-verify-kiosk.sh` (nginx:alpine +
hostPath dist) as a stopgap. This chart is the production artifact for once the
image is pullable (or mirror it into the `uneeq` project and override `image.*`).

`ingress.routes.asr` is **off by default** — turn it on once `riva-ws-proxy`
(same Harbor project) is running.
