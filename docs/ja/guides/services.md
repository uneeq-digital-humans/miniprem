# MiniPremサービス概要

MiniPremプラットフォームは、包括的なデジタルヒューマン体験を提供するために連携するいくつかの統合サービスで構成されています。このガイドでは、これらのサービスとその相互作用の概要を提供します。

## コアサービス

| サービス | 目的 | ポート | ドキュメント |
|---------|---------|------|---------------|
| Renny | デジタルヒューマンアバター | 8081 | [Rennyガイド](renny.md) |
| vLLM | 大規模言語モデル | 8000 | [vLLMガイド](vllm.md) |
| Flowise | ワークフロー自動化 | 3000 | [Flowiseガイド](flowise.md) |
| Redis | キュー管理 | 6379 | - |
| Prometheus | メトリクス収集 | 9090 | [監視ガイド](monitoring.md) |
| Grafana | メトリクス可視化 | 3001 | [監視ガイド](monitoring.md) |
| Audio2Face | 顔のアニメーション | 50000, 52000 | [Rennyガイド](renny.md) |
| RIME | テキストから音声へのAPI | 8100 | [RIMEガイド](rime.md) |
| Whisper | 音声認識 | 9000 | [Whisperガイド](whisper.md) |

## サービスの依存関係

- **Renny**は以下に依存します:
  - 顔のアニメーションのためのAudio2Faceサービス
  - テキストから音声への外部サービス（Azure Speech Services）
  - アバターレンダリングのためのUneeQプラットフォーム（外部）

- **Flowise**は以下に依存します:
  - 言語モデル機能のためのvLLM
  - キュー管理のためのRedis
  - データベースストレージのためのSQLite（組み込み）

- **監視**は以下に依存します:
  - メトリクス収集のためのPrometheus
  - 可視化のためのGrafana

## 環境変数

各サービスは、Docker Composeファイルの環境変数を通じて構成されます。主要な環境変数には、次のものがあります:

- **Renny**:
  - `DHOP_ADDRESS`: UneeQプラットフォームアドレス
  - `A2F_ADDRESS`: Audio2Faceサービスアドレス
  - `AZURE_REGION`および`AZURE_SPEECH`: スピーチサービス資格情報

- **Flowise**:
  - `DATABASE_TYPE`: ローカルデータベースに設定（SQLite）
  - `FLOWISE_USERNAME`および`FLOWISE_PASSWORD`: 認証資格情報
  - `REDIS_HOST`および`REDIS_PORT`: Redis接続の詳細

- **vLLM**:
  - `NVIDIA_VISIBLE_DEVICES`: モデル推論のためのGPU割り当て

## ボリューム

永続データはDockerボリュームに保存されます:

- **vllm_data**: ダウンロードされた言語モデルを保存します
- **flowise_data**: Flowiseの設定とデータベースを保存します
- **redis_data**: Redisキュー データを保存します
- **prometheus_data**: メトリクスの履歴を保存します
- **grafana_data**: ダッシュボードの設定を保存します

## ネットワーク構成

ほとんどのサービスは、通信のためにデフォルトのDockerネットワークを使用しますが、以下の例外があります:

- **Renny**は、最適なパフォーマンスのために`network_mode: "host"`を使用します
- サービスは、Dockerネットワーク内でコンテナ名を使用して相互に参照します（例：`http://vllm:8000`）

## サービスのヘルスチェック

すべてのサービスには、正常に機能していることを確認するためのヘルスチェックが含まれています:

```yaml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:PORT/health"]
  interval: 10s
  timeout: 5s
  retries: 3
```

これらのヘルスチェックは、サービス起動の依存関係を調整するために使用されます。
