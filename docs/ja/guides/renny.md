# Rennyデジタルヒューマン

このガイドでは、MiniPremプラットフォームのRennyデジタルヒューマンコンポーネントについて説明します。これは、人間のようなインタラクションのためのビジュアルインターフェースを提供します。

## 概要

Rennyは、UneeQのテクノロジーを搭載したデジタルヒューマンアバターで、AIインタラクションのためのビジュアルインターフェースを提供します。顔のアニメーション、唇の同期、ジェスチャー機能を組み合わせて、より魅力的な会話体験を作り出します。

## Rennyへのアクセス

- **ヘルスエンドポイント**: http://localhost:8081/health
- **コンテナ名**: `renny`

## アーキテクチャ

Rennyコンポーネントは、他のいくつかのサービスと連携します:

1. **Audio2Face統合**: 音声を顔のアニメーションに変換します
2. **UneeQプラットフォーム**: デジタルヒューマンのレンダリングを管理します
3. **Azure Speech Services**: テキストから音声への機能を提供します

## 設定

### メイン設定ファイル

Rennyの主な設定は、`docker/configuration.dat`に保存され、次のものが含まれます:

- **サーバー**: UneeQサーバーのエンドポイント
- **TenantId**: UneeQテナント識別子
- **JWSSecret**: UneeQサービス用の認証トークン

### 環境変数

`docker/docker-compose.env`の主要な環境変数:

- **A2F_ADDRESS**: Audio2Faceサービスアドレス
- **DHOP_ADDRESS**: UneeQプラットフォームアドレス
- **DHOP_APIKEY**: UneeQプラットフォームAPIキー
- **DHOP_TENANTID**: UneeQテナントID
- **AZURE_REGION**: スピーチサービス用のAzureリージョン
- **AZURE_SPEECH**: Azureスピーチサービスキー

## ヘルスマネジメント

Rennyのヘルスステータスは、次のコマンドで確認できます:

```bash
curl -f http://localhost:8081/health
```

このエンドポイントは、サービスの現在の状態と依存サービスへの接続に関する情報を返します。

## ネットワーク構成

Rennyは、最適なパフォーマンスを確保するためにホストネットワークモードを使用します:

```yaml
network_mode: "host"
```

これにより、RennyはDockerネットワーク分離を使用せずにシステムネットワークインターフェイスに直接アクセスできます。

## GPUアクセラレーション

Rennyは、レンダリングにNVIDIA GPUアクセラレーションを活用します:

```yaml
runtime: nvidia
```

これにより、アニメーションと顔の表情がスムーズになります。

## LLMとの統合

RennyとLLM（Flowise経由）の統合は、次のように機能します:

1. ユーザー入力がキャプチャされます（テキストまたは音声）
2. 入力はFlowise/vLLMパイプラインによって処理されます
3. 応答はAzure TTSを介して音声に変換されます
4. Audio2Faceは、音声に同期した顔のアニメーションを生成します
5. Rennyは、応答を話すアニメーション化されたアバターをレンダリングします

## 詳細なカスタマイズ

### レンダリングオプション

Rennyは、コマンドライン引数を使用してさまざまなレンダリング構成をサポートしています:

```
-RenderOffScreen  # ヘッドレスレンダリング
-ResX=1920        # 水平解像度
-ResY=1080        # 垂直解像度
```

グラフィカル表示（ヘッドレスではなく）、Docker構成を変更できます:

```yaml
# ビジュアルレンダリングのためにコメントを解除します
environment:
  - DISPLAY=$DISPLAY
volumes:
  - /tmp/.X11-unix:/tmp/.X11-unix
  - ~/.Xauthority:/home/ue4/.Xauthority
```

### アニメーション設定

Audio2Faceアニメーションパラメータは、A2F構成ファイルで調整できます:

- **唇の同期**: 口の動きの精度を制御します
- **表情の強度**: 顔の表情の強さを調整します
- **まばたきパラメータ**: まばたきの頻度とスタイルを制御します

## トラブルシューティング

### 一般的な問題

1. **ビジュアル出力なし**:
   - `-RenderOffScreen`が有効になっているかどうかを確認します
   - GPUドライバーとレンダリング機能を確認します

2. **アニメーションの品質が悪い**:
   - A2Fサービスのヘルスを確認します
   - 音声の品質と処理を確認します

3. **接続の問題**:
   - UneeQプラットフォームの接続を確認します
   - ネットワーク設定とファイアウォールルールを確認します

4. **オーディオとビジュアルの同期の問題**:
   - `A2F_AUDIO_DELAY_TIME_MS`パラメータを調整します
   - システムパフォーマンスを確認して、レンダリングの遅延がないかを確認します
