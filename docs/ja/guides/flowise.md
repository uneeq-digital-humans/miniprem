# Flowise 設定ガイド

Flowiseは、AIワークフローの作成と管理のためのビジュアルインターフェースを提供します。このガイドは、FlowiseをvLLM統合で設定して使用するのに役立ちます。

## Flowiseへのアクセス

- **URL**: http://localhost:3000
- **デフォルトの資格情報**:
  - ユーザー名: `user`
  - パスワード: `password`

## Flowiseの概念

- **チャットフロー**: 会話ワークフローのビジュアル表現
- **ノード**: 特定の機能を実行するコンポーネント（プロンプト、モデル、メモリなど）
- **エッジ**: ノード間の接続で、データの流れを定義します
- **API**: チャットフローとプログラムで対話するためのRESTfulエンドポイント

## vLLMを使用したチャットフローの作成

### 1. Flowise UIにアクセス

1. ブラウザを開き、http://localhost:3000に移動します
2. ユーザー名`user`とパスワード`password`でログインします

### 2. 新しいチャットフローを作成

1. サイドバーで「Chatflows」をクリックします
2. 新しいチャットフローを作成するには、「+」ボタンをクリックします
3. チャットフローの名前を付けます（例： "vLLM Gemma3 Chatflow"）

### 3. ノードを追加して構成する

#### システムプロンプトノード

1. ノードパネルから、「システムプロンプト」ノードをキャンバスにドラッグアンドドロップします
2. ノードを次のように構成します:
   - プロンプト: "Gemma3を搭載したヘルプフルアシスタントです。簡潔で正確な回答を提供してください。"

#### vLLMノード

1. ノードパネルから、「vLLM」または「OpenAI互換」ノードをキャンバスにドラッグアンドドロップします
2. ノードを次のように構成します:
   - ベースURL: `http://vllm:8000/v1`（Dockerコンテナ名を使用。localhostは使用しない）
   - モデル: `gemma-3-4b`
   - 温度: `0.7`
   - 最大トークン数: `1000`
   - 他の設定はデフォルト値のままにします

#### バッファメモリノード

1. ノードパネルから、「バッファメモリ」ノードをキャンバスにドラッグアンドドロップします
2. ノードを次のように構成します:
   - メモリキー: `chat_history`
   - メッセージを返す: `true`（チェック）
   - 最大トークン制限: `2000`

#### 会話チェーンノード

1. ノードパネルから、「会話チェーン」ノードをキャンバスにドラッグアンドドロップします
2. 追加の構成は必要ありません

#### チャットトリガーノード

1. ノードパネルから、「チャットトリガー」ノードをキャンバスにドラッグアンドドロップします
2. 追加の構成は必要ありません

### 4. ノードを接続する

ノードを以下のように接続します:

1. システムプロンプト → 会話チェーン（「プロンプト」から「systemPrompt」へ）
2. vLLM → 会話チェーン（「モデル」から「llm」へ）
3. バッファメモリ → 会話チェーン（「メモリ」から「memory」へ）
4. 会話チェーン → チャットトリガー（「出力」から「入力」へ）

### 5. 保存してテストする

1. 右上の「保存」ボタンをクリックします
2. チャットフローをテストするには、「チャット」ボタンをクリックします

## Flowise APIの使用

Flowise APIを使用してチャットフローと対話できます。

### 認証

APIリクエストに以下のヘッダーを追加します:
```
  "Authorization: Bearer YOUR_DEFAULT_TOKEN_HERE"
```
