# 플로우이즈 구성 가이드

Flowise는 AI 워크플로우를 생성하고 관리하기 위한 시각적 인터페이스를 제공합니다. 이 가이드는 vLLM 통합과 함께 Flowise를 설정하고 사용하는 데 도움이 됩니다.

## Flowise 액세스하기

- **URL**: http://localhost:3000
- **기본 자격 증명**:
  - 사용자 이름: `사용자`
  - 비밀번호: `password`

## 플로우이즈 개념

- **대화 흐름**: 대화 워크플로우의 시각적 표현
- 노드**: 특정 기능을 수행하는 구성 요소(프롬프트, 모델, 메모리 등)
- 에지**: 데이터 흐름 방식을 정의하는 노드 간 연결
- **API**: 프로그래밍 방식으로 채팅 흐름과 상호 작용하는 RESTful 엔드포인트

## vLLM으로 채팅 플로우 만들기

### 1. 플로우이즈 UI에 액세스

1. 브라우저를 열고 다음 주소로 이동합니다: http://localhost:3000
2. 사용자 이름 `user`와 비밀번호 `password`로 로그인합니다.

### 2. 새 채팅 플로우 만들기

1. 사이드바에서 "챗플로우"를 클릭합니다.
2. "+" 버튼을 클릭하여 새 채팅 플로우를 만듭니다.
3. 채팅플로우의 이름을 지정합니다(예: "vLLM Gemma3 Chatflow").

### 3. 노드 추가 및 구성

#### 시스템 프롬프트 노드

1. 노드 패널에서 "시스템 프롬프트" 노드를 캔버스로 끌어다 놓습니다.
2. 다음을 사용하여 노드를 구성합니다:
   - 프롬프트: "귀하는 Gemma3로 구동되는 유용한 어시스턴트입니다. 간결하고 정확한 응답을 제공하세요."

#### vLLM 노드

1. 노드 패널에서 "vLLM" 또는 "OpenAI 호환" 노드를 캔버스로 끌어다 놓습니다.
2. 다음을 사용하여 노드를 구성합니다:
   - 기본 URL: `http://vllm:8000/v1`(로컬 호스트가 아닌 도커 컨테이너 이름 사용)
   - 모델: `gemma-3-4b`
   - 온도: `0.7`
   - 최대 토큰: `1000`
   - 다른 설정은 기본값으로 두세요.

#### 버퍼 메모리 노드

1. 노드 패널에서 "버퍼 메모리" 노드를 캔버스로 끌어다 놓습니다.
2. 다음을 사용하여 노드를 구성합니다:
   - 메모리 키: 'chat_history`
   - 메시지 반환: 참` (체크됨)
   - 최대 토큰 제한: `2000`

#### 대화 체인 노드

1. 노드 패널에서 "대화 체인" 노드를 캔버스로 끌어다 놓습니다.
2. 추가 구성이 필요하지 않습니다.

#### 채팅 트리거 노드

1. 노드 패널에서 '채팅 트리거' 노드를 캔버스로 끌어다 놓습니다.
2. 추가 구성이 필요하지 않습니다.

### 4. 노드 연결하기

다음 연결을 사용하여 노드를 연결합니다:

1. 시스템 프롬프트 → 대화 체인("프롬프트"에서 "시스템 프롬프트"로)
2. vLLM → 대화 체인("model"에서 "llm"으로)
3. 버퍼 메모리 → 대화 체인("메모리"에서 "메모리"로)
4. 대화 체인 → 채팅 트리거 ("출력"에서 "입력"으로) 4.

### 5. 저장 및 테스트

1. 오른쪽 상단의 '저장' 버튼을 클릭합니다.
2. "채팅" 버튼을 클릭하여 채팅 플로우를 테스트합니다.

## 플로우이즈 API 사용하기

플로우이즈 API를 통해 채팅 플로우와 상호작용할 수 있습니다.

### 인증

API 요청에 다음 헤더를 추가하세요:
```
  "Authorization: Bearer YOUR_DEFAULT_TOKEN_HERE"
```
