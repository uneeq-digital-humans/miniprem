# 미니프렘 서비스 개요

미니프렘 플랫폼은 종합적인 디지털 휴먼 경험을 제공하기 위해 함께 작동하는 여러 통합 서비스로 구성되어 있습니다. 이 가이드에서는 이러한 서비스에 대한 개요와 상호 작용 방식에 대해 설명합니다.

## 핵심 서비스

| 서비스 | 목적 | 포트 | 설명서 |
|---------|---------|------|---------------|
| Renny | 디지털 휴먼 아바타 | 8081 | [Renny Guide](renny.md) |
| vLLM | 대규모 언어 모델 | 8000 | [vLLM 가이드](vllm.md) |
| Flowise | 워크플로우 자동화 | 3000 | [Flowise 가이드](flowise.md) |
| Redis | 대기열 관리 | 6379 | - |
| Prometheus | 메트릭 수집 | 9090 | [모니터링 가이드](monitoring.md) |
| Grafana | 메트릭 시각화 | 3001 | [모니터링 가이드](모니터링.md) |
| Audio2Face | 얼굴 애니메이션 | 50000, 52000 | [레니 가이드](renny.md) |
| RIME | 텍스트 음성 변환 API | 8100 | [RIME 가이드](rime.md) |
| 위스퍼 | 음성 인식 | 9000 | [위스퍼 가이드](whisper.md) |

서비스 종속성 ## 서비스 종속성

- 레니**는 다음과 같이 의존합니다:
  - 얼굴 애니메이션을 위한 Audio2Face 서비스
  - 텍스트 음성 변환을 위한 Azure 음성 서비스(외부)
  - 아바타 렌더링을 위한 UneeQ 플랫폼(외부)

- Flowise**는 다음을 필요로 합니다:
  - 언어 모델 기능을 위한 vLLM
  - 대기열 관리를 위한 Redis
  - 데이터베이스 저장용 SQLite(임베디드)

- 모니터링**은 다음에 의존합니다:
  - 메트릭 수집을 위한 Prometheus
  - 시각화를 위한 Grafana

## 환경 변수

각 서비스는 Docker Compose 파일의 환경 변수를 통해 구성됩니다. 주요 환경 변수는 다음과 같습니다:

- 레니**:
  - dhop_address`: UneeQ 플랫폼 주소
  - a2f_address`: Audio2Face 서비스 주소
  - `azure_region` 및 `azure_speech`: 음성 서비스 자격 증명

- 플로우이즈**:
  - 데이터베이스 유형`: 로컬 데이터베이스의 경우 SQLite로 설정
  - 플로우이즈_사용자명` & `플로우이즈_비밀번호`: 인증 자격 증명
  - redis_host` & `redis_port`: Redis 연결 세부 정보

- **vLLM**:
  - `nvidia_visible_devices`: 모델 추론을 위한 GPU 할당

## 볼륨

퍼시스턴트 데이터는 도커 볼륨에 저장됩니다:

- vllm_data**: 다운로드한 언어 모델을 저장합니다.
- 플로우이즈_데이터**: Flowise 구성 및 데이터베이스를 저장합니다.
- **redis_data**: Redis 큐 데이터를 저장합니다.
- 프로메테우스_데이터**: 메트릭 기록을 저장합니다.
- **grafana_data**: 대시보드 구성을 저장합니다.

## 네트워크 구성

이러한 예외를 제외하고 대부분의 서비스는 통신을 위해 기본 Docker 네트워크를 사용합니다:

- Renny**는 최적의 성능을 위해 `network_mode: "host"`를 사용하여 최적의 성능을 제공합니다.
- 서비스는 Docker 네트워크 내에서 컨테이너 이름(예: `http://vllm:8000`)으로 서로를 참조합니다.

## 서비스 상태 확인

모든 서비스에는 제대로 작동하는지 확인하기 위한 상태 점검이 포함되어 있습니다:

```yaml
healthcheck:
  test: ["CMD", "curl", "-f", "http://localhost:PORT/health"]
  interval: 10s
  timeout: 5s
  retries: 3
```

이러한 상태 확인은 서비스 시작 종속성을 조정하는 데 사용됩니다.
